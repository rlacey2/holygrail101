VIEW 1 Holy Grail Layout

All headers/footers hidden => View 2
A Holy Grail effort on steroids, the holy grail refers to a web page layout which has multiple, equal height columns that are defined with style sheets. It is commonly desired and implemented, although the ways in which it has tended to be implemented all have drawbacks. Consequently, finding an optimal implementation has been compared to searching for the elusive Holy Grail.

The approach here is to try and produce a responsive application for mobile to desktop using flex and bootstrap 4. The primary intended target device is desktop-level, with the bonus of availability on a small device (<768px)

There is a global application header and footer with a component level main body. A dynamic [ngStyle] is used to adjust top/bottom if these are not displayed.

The component level which fills the main body is essentially the holy grail, 3 columns of equal height. Each component has its own header and 3 equal length columns: a left side, main central content body for the component and a right side.

The header and asides are intended to hold functional navigation/action/features relative to the component view.

A lot of the heavy lifting is done here, further styling is left as an exercise for the reader/user to adjust position of relevant header/footer items. Delete unneeded elements and becareful of z-index values for overlapping areas with dropdown features on smaller displays.

Lots of colour used to show the various areas of the layout. Elements can be now added or removed to adjust the layout to a practial need.
